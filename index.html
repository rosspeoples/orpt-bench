OpenCode benchmark publication

Benchmark data you can actually compare at a glance.

ORPT-Bench measures correctness, request efficiency, time, and cost across a fixed task suite. This publication is the operator view: rankings, per-task deltas, cost and speed tradeoffs, catalog context, and links back to the raw benchmark artifacts.

Hint: every comparison table is sortable and starts in composite-score order.
How to read the ranking
Composite score is 70% raw task score and 30% value score. Value score blends ORPT, total cost, and wall time using the published scoring weights.
What lives here
Model leaderboard tables, task-by-task comparison grids, benchmarked model catalog cards, embedded charts, docs links, and historical snapshots.
Where deeper detail lives
Use the docs section for the benchmark design, schema, model catalog, raw JSON, and history index. Every embedded chart also has a direct artifact link.
Research framing
Artificial Analysis style depth comes from making tradeoffs visible: benchmark-composition context, category and difficulty splits, pairwise deltas, and observed-versus-prior cost/speed comparisons.
